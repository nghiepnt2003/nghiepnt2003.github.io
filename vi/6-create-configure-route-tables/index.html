Tạo và cấu hình Route Tables

Route Tables là thành phần quan trọng trong AWS VPC, giúp điều hướng lưu lượng mạng giữa các subnet cũng như giữa VPC với internet. Trong phần này, bạn sẽ tạo và cấu hình các Route Table để kết nối Public và Private Subnets.

Bước 1: Tạo Route Table cho Public Subnet

  1. Truy cập dịch vụ VPC:

    • Truy cập vào VPC Dashboard trong AWS Management Console hoặc gõ “VPC” trên thanh tìm kiếm.

  2. Chọn Route Tables:

    • Trong VPC Dashboard, chọn Route Tables từ thanh điều hướng bên trái.

  3. Nhấn nút Create Route Table:

    • Nhấn Create route table để tạo Route Table mới.

  4. Cấu hình Route Table:

    • Name: Đặt tên cho Route Table (ví dụ: my-route-table).
    • VPC: Chọn VPC mà bạn đã tạo ở phần trước.

  5. Nhấn nút Create route table để hoàn tất.

Bước 2: Cấu hình Route cho Public Subnet

  1. Chọn Route Table vừa tạo:

    • Chọn Route Table mà bạn vừa tạo (ví dụ: PublicRouteTable).

  2. Thêm route tới Internet Gateway:

    • Chuyển đến tab Routes và nhấn Edit routes.
    • Nhấn Add route và cấu hình như sau:
      • Destination: Nhập 0.0.0.0/0 (đại diện cho toàn bộ địa chỉ IP trên internet).
      • Target: Chọn Internet Gateway (IGW) mà bạn đã tạo trước đó từ danh sách.

  3. Lưu cấu hình:

    • Nhấn Save routes để lưu cấu hình Route Table.

  4. Liên kết với Public Subnet:

    • Chuyển sang tab Subnet associations và nhấn Edit subnet associations.

    • Chọn Public Subnet của bạn từ danh sách và nhấn Save associations để liên kết Route Table với Public Subnet.

Bước 3: Tạo Route Table cho Private Subnet

  1. Nhấn Create Route Table:

    • Tạo thêm một Route Table mới cho Private Subnet (ví dụ: PrivateRouteTable).

  2. Không thêm Internet Gateway cho Private Subnet:

    • Trong Route Table của Private Subnet, không thêm route 0.0.0.0/0 trỏ tới Internet Gateway, để đảm bảo rằng Private Subnet không thể kết nối trực tiếp với internet.

  3. Liên kết với Private Subnet:

    • Tương tự như bước trên, trong tab Subnet associations, chọn Private Subnet và nhấn Save associations.

Kết quả

  • Public Subnet: Đã được gắn Route Table với route đến Internet Gateway, cho phép EC2 instances trong Public Subnet có thể truy cập internet.
  • Private Subnet: Có Route Table riêng, không được phép truy cập trực tiếp internet, đảm bảo tính bảo mật cho các tài nguyên bên trong.
